# Deep code audit — July 2026 (v0.5.0)

A full security / reliability / performance / quality audit of the daemon and board,
run as a multi-agent effort (parallel Claude + Codex implementers, with cross-provider
**adversarial review** of every change) and landed on branch `fd/audit-cleanup` (released as v0.5.0, 2026-07-13).

## What was fixed

**Security**
- **CSRF / cross-site WebSocket hijack (critical).** Loopback was auto-trusted with no
  `Origin`/`Host`/`Content-Type` check, so any visited web page could read the `/ws`
  snapshot, drive `/ws/term`, and POST `/api/spawn`. Added an `Origin`/`Host`/`Sec-Fetch`
  allowlist + `application/json` requirement on all state-changing routes and both WS
  upgrades (also closes DNS rebinding), **without** breaking the zero-config loopback board.
  Mutating GET routes (`/mail`, `/api/watch`) are gated too.
- Token no longer rides the WS broadcast (kept on token-gated `/state`); token no longer
  logged; `fleetd.log` and hook-payload capture are `0600` (capture off by default);
  `remote.url` restricted to `https://claude.ai`; mDNS instance name anonymized.

**Reliability / correctness**
- Worktree removal no longer `rm -rf`s uncommitted work without `--force`.
- Boot reconciliation no longer tombstones the live fleet on a tmux timeout.
- Per-paste unique tmux buffer (no cross-agent mail bleed); revive/kill hardened
  (stale-id can't kill a revived pane; provisional row + compensation; single-flight).
- Byte-correct UTF-8 request bodies; guarded conflict parse; WS keepalive + backpressure;
  bounded DB/caches; port-aware pid guard; global `unhandledRejection` handler.

**Four production bugs** (reported over the fleet board after they lost every live terminal):
1. `/clear` (`SessionEnd reason='clear'`) no longer condemns a live pane.
2. Retention verifies the pane via tmux before presuming a silent-but-live agent dead.
3. `pane-dead`/`gone` spawns are re-validated and **resurrected** (with hysteresis); `revive()`
   adopts a live pane instead of 409-deadlocking; the one-way door is closed.
4. Mail no longer silently truncates at 500 chars (bounded 4000, surrogate-safe, signalled).

**Performance** — broadcast coalescing, tiered transcript reads, memoized board + clock
context, idle poll backoff, batched PTY output, prepared-statement cache.

**Structure** — `derive.mjs` (2955 lines) split into a 314-line composition root + 13 focused
modules (`helpers`, `statements`, `worktrees`, `mail`, `ledger`, `ingest`, `commands`,
`plans`, `spawns`, `events`, `snapshot`, `retention`, `env-scrub`); `App.jsx` (763 → 433)
split into hooks + a `Header` component. Both verified pure, behavior-preserving.

Dead code and duplication removed throughout. Accessibility: ARIA-legal cards, modal focus
traps, keyboard tile focus.

## Deferred minor cleanups (safe follow-ups, not done here)

Low-ROI, cross-file micro-dedups intentionally left to keep the release focused and low-risk:

- **D3** — four subprocess wrappers (`execFileP`, `tmux()`, `git()`, `runOnce()`) with three
  failure conventions could share one result shape in an `exec.mjs`.
- **D4** — `envInt` is duplicated in `helpers.mjs` + `termbridge.mjs` (+ an `envMs` variant in
  `fleet-watch.mjs`); could be one shared env helper.
- **D5** — `FLEETDECK_HOME`/`PORT`/`BASE` resolution is triplicated across `fleetd.mjs`,
  `fleet-sessionstart.mjs`, `fleet-watch.mjs`; could be one `config.mjs`.
- **D6** — `termbridge.mjs` re-derives the scoped tmux `sessionName`/`windowName` by hand
  instead of importing them from `spawn.mjs`.
- **D11** — the `/state` snapshot serializes pending-mail three ways (`mail_pending` ×2 +
  `mail_meta`); `mail_meta` supersedes the others and they could be consolidated once the
  board drops the legacy fields.
